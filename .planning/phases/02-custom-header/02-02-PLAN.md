  Nav pill display name lookup:
    i18n(`filters.${name}.title`)   — for count=0 case

  Relevant keys: filters.latest.title, filters.top.title, filters.hot.title, filters.categories.title

CRITICAL: filters.latest.title is a PLURALIZED KEY — it is an object { zero, one, other }, NOT a plain string.

The complete updated file should be:

```javascript
import { apiInitializer } from "discourse/lib/api";
import { i18n } from "discourse-i18n";
import TavernBanner from "../components/tavern-banner";

export default apiInitializer("1.13.0", (api) => {
  // Phase 1: render homepage banner
  if (settings.show_homepage_banner) {
    api.renderInOutlet("discovery-list-container-top", TavernBanner);
  }

  // Phase 2: rename nav pill labels and Sign In button text
  // NOTE: theme locales/en.yml cannot override js.* core strings (theme locale namespace
  // is isolated). JS patch is the correct approach for an English-only forum.
  // top_menu site setting must be set to "hot|latest|categories|top" via Admin → Settings
  // for the Trending (/hot) pill to appear (top_menu is not a themeable setting).
  const locale = i18n.currentLocale();
  const translations = i18n.translations[locale];
  if (translations?.js) {
    const { filters } = translations.js;
    if (filters) {
      // filters.latest.title is a pluralized key — must assign an object, not a string
      if (filters.latest) {
        filters.latest.title = {
          zero: "Latest at the Bar",
          one: "Latest at the Bar (%{count})",
          other: "Latest at the Bar (%{count})"
        };
      }
      if (filters.top) {
        filters.top.title = "Top Shelf";
      }
      if (filters.hot) {
        filters.hot.title = "Trending";
      }
      if (filters.categories) {
        filters.categories.title = "Rooms";
      }
    }
    // Rename "Log In" button label to "Sign In"
    if (translations.js.log_in !== undefined) {
      translations.js.log_in = "Sign In";
    }
  }
});
```

Key constraints:
- The `show_homepage_banner` guard was moved to wrap ONLY the banner renderInOutlet call — the i18n patch runs unconditionally (it should always apply regardless of banner setting)
- The `filters.latest.title` assignment MUST be an object with `zero`, `one`, `other` keys — assigning a plain string will cause "[missing translation]" errors in the console
- All null/undefined guards (`?.js`, `?.filters`, `if (filters.latest)`) are required — the translations object structure may vary on different Discourse versions
- Do NOT use `locales/en.yml` for any of these keys — theme locale files cannot override `js.*` namespace
