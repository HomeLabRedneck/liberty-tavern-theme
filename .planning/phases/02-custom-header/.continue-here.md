# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: GJS strict mode — `theme-asset` helper is NOT in scope in `.gjs` strict mode templates. Any connector using `theme-asset` must use `.hbs` OR access the value via a JS class getter (`settings.logo`). The tavern-logo connector was broken twice by this.
- [ ] CONSTRAINT: CSS nav pills hack is route-scoped — `.navigation-container .nav-pills` with `position: fixed` only renders on discovery routes. Non-discovery pages (Subscribe, topic pages, admin) show no pills. The full fix is on `feature/persistent-header-nav` branch.
- [ ] CONSTRAINT: Discourse `!important` arms race — Discourse's default styles often require `!important` overrides. Adding `!important` is acceptable but creates brittleness. When Discourse updates selectors, our overrides silently stop working.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| theme-asset in GJS | Using `{{theme-asset "file"}}` in `.gjs` strict mode templates causes compile error "not in scope" | blocking | Always use `.hbs` for connectors that need `theme-asset`, OR use `settings.logo` (upload type) as the source |
| .navigation-container position:fixed | CSS hack moves nav pills to header but only on discovery routes; breaks on plugin pages | advisory | Use `api.headerIcons.add()` with GJS component (see feature/persistent-header-nav branch) |
| Conflicting nav active-state blocks | Two SCSS blocks define nav active state differently; later block wins silently | advisory | Single source of truth for nav pill styles; check for duplicate selectors before adding new overrides |
