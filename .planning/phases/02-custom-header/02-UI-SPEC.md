## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Header Usage |
|-------|-------|--------------|
| xs | 4px | Logo image-to-text gap within `.tavern-logo` |
| sm | 8px | Nav pill horizontal padding (each side) |
| md | 16px | Logo area left padding; nav item gap |
| lg | 24px | Header total vertical padding (top + bottom combined) |
| xl | 32px | Gap between logo block and nav block |
| 2xl | 48px | Not used in header |
| 3xl | 64px | Not used in header |

Exceptions:
- Header total height target: 64px (Discourse default is ~60px; 64px aligns to 8-point grid and provides comfortable touch target for icons).
- Sign In button: 32px height (8px top + 8px bottom padding) to clear the 44px touch target via surrounding header padding — the button itself need not be 44px tall since it sits in a 64px header bar.
- Logo image: 36px height (not a spacing token — a fixed asset dimension). Width scales proportionally.

Source: existing `.d-header` SCSS section + Image 1 design target proportions.

## Color

| Role | Value | Header Usage |
|------|-------|--------------|
| Dominant (60%) — dark header surface | `#1a120c` (`--header_background`) | `.d-header` background — the entire header bar |
| Secondary (30%) — cream text | `#f5ebd9` (`--header_primary`, `--tavern-cream`) | Logo title, tagline, nav links (default state) |
| Accent (10%) — brass | `var(--tavern-brass)` (≈ `#c8941a`) | Sign In button background; `.d-header` bottom border; nav link hover color; logo tagline separator dot color |
| Oxblood | `var(--tavern-oxblood)` (`#7a1f1f`) | Not used in Phase 2 header elements |
| Destructive | `#9a2a1a` (`--danger`) | Not applicable — no destructive actions in header |

**Accent (`--tavern-brass`) is reserved for exactly these header elements:**
1. Sign In button background (`.d-header .login-button` background)
2. Header bottom border (`border-bottom: 1px solid var(--tavern-brass)` — already in place)
3. Nav link hover state color (`color: var(--tavern-brass)` on `:hover`)
4. Search icon and user-menu icon hover state (already in place via `.d-header .nav-item-link:hover`)

**Accent is NOT applied to:** logo title, tagline text, nav link default state, header background, or any icon default state.

Nav link default state: `#f5ebd9` at 0.85 opacity (already established in `common.scss` line 55–57).
Nav link active state: `#f5ebd9` at full opacity (1.0), plus a 2px underline in `var(--tavern-brass)` below the pill.
Nav link hover state: `var(--tavern-brass)` color, opacity 1.0.

Source: `about.json` color_schemes + `common.scss` `.d-header` block.

## Component Contracts

### Logo Connector: `connectors/home-logo-contents/tavern-logo.hbs`

**Layout:** Horizontal flex row. Logo image flush left, title + tagline stacked vertically to the right of the image.

```
[ img 36px tall ]  [ .tavern-logo__title   ] ← 20px Playfair Display italic 900
                   [ .tavern-logo__tagline ] ← 10px Inter 700 uppercase 0.18em spacing
```

**BEM structure:**
```
.tavern-logo                     ← flex container, align-items: center, gap: 8px (sm)
  img.tavern-logo__image          ← height: 36px; width: auto; display: block
  .tavern-logo__text              ← flex column, justify-content: center
    .tavern-logo__title           ← 20px Playfair Display italic 900, color: #f5ebd9, line-height: 1.1
    .tavern-logo__tagline         ← 10px Inter 700 uppercase, letter-spacing: 0.18em, color: #f5ebd9, opacity: 0.7, line-height: 1.2
```

**HBS template contract:**
- Image source: `{{theme-asset 'logo.png'}}` — no JS file, no Glimmer class.
- No inline `style=""` attributes — all styling via `common.scss` BEM classes.
- No `<a>` wrapping needed; Discourse wraps the entire `home-logo-contents` outlet in a link to the home route.

**Responsive behavior:** No change at mobile breakpoints. Discourse's native header handles mobile collapse. Do not add `@media` rules to the logo connector CSS.

---

### Nav Links: Discourse Native `.nav-pills`

Nav links are rendered by Discourse natively via `top_menu` site setting. Phase 2 applies only I18n label overrides and SCSS styling — no custom component.

**States:**

| State | Color | Opacity | Decoration |
|-------|-------|---------|------------|
| Default | `#f5ebd9` | 0.85 | none |
| Hover | `var(--tavern-brass)` | 1.0 | none |
| Active (current route) | `#f5ebd9` | 1.0 | 2px bottom border in `var(--tavern-brass)` |
| Focus (keyboard) | `#f5ebd9` | 1.0 | 2px outline in `var(--tavern-brass)`, offset 2px (do not use `outline: none`) |

Already partially established: `common.scss` lines 54–57 style `.nav-item-link` default and hover states. Phase 2 adds the active underline rule and the focus visible rule.

**SCSS target:** `.d-header .nav-pills .nav-item a` and `.d-header .nav-pills .nav-item.active a` — do NOT use `display`, `flex-direction`, or `position` on `.d-header` itself (CLAUDE.md critical rule #3 / SUMMARY.md pitfall #3).

---

### Sign In Button: `.d-header .login-button`

Discourse renders `.login-button` natively for anonymous users. Phase 2 applies SCSS only.

**States:**

| State | Background | Color | Border | Effect |
|-------|------------|-------|--------|--------|
| Default | `var(--tavern-brass)` | `#1a120c` | none | none |
| Hover | `var(--tavern-brass)` | `#1a120c` | none | `filter: brightness(1.15)` |
| Focus (keyboard) | `var(--tavern-brass)` | `#1a120c` | 2px outline `#f5ebd9` offset 2px | none |
| Active (click) | `var(--tavern-brass)` | `#1a120c` | none | `filter: brightness(0.9)` |

**Typography:** 11px Inter 700, uppercase, letter-spacing 0.12em. Border-radius: 2px. (Matches `.sign-up-button` pattern already in `common.scss` lines 60–71 — extend that block to include `.login-button`.)

**Visibility rule:** Do not add `display: none` or visibility overrides. Discourse natively shows `.login-button` only to anonymous users and hides it for authenticated users. Do not interfere with this behavior.

---

### Native Icons: Search + User Menu

**Contract:** Do not hide, reposition, or resize `.d-header-icons .search-button` or `.d-header-icons .hamburger-button` or `.d-header-icons .current-user`. Their default state color (`#f5ebd9`) and hover (`var(--tavern-brass)`) are already set via `common.scss` line 54–57.

Phase 2 adds no new rules for these icons — HEAD-05 (preserve native search and user-menu) is satisfied by the existing rules combined with not overriding `.d-header` layout properties.

---

### Header Height

Target: 64px total height.

Implementation: Set `min-height: 64px` on `.d-header`. Do NOT set `height` (use `min-height` to allow content-driven expansion on narrow viewports). Do NOT set `display`, `flex-direction`, or `position` on `.d-header`.

Add compensating `padding-top` on `#main-outlet` or `#main-outlet-wrapper` equal to the header height to prevent content jumping under the fixed header: `padding-top: 64px`.

## Implementation Constraints (from CLAUDE.md — mandatory)

These are hard rules for the executor — violations have been reverted multiple times:

1. No `common/header.html` — HTML injection in the header is forbidden. Use `home-logo-contents` outlet + SCSS.
2. No `display`, `flex-direction`, or `position` overrides on `.d-header` — only colors, fonts, borders, height/min-height.
3. No inline `style=""` attributes in `tavern-logo.hbs` — all styling in `common.scss`.
4. No `@import` for Google Fonts — fonts load once via `head_tag.html`; do not add any `@import` in SCSS.
5. `top_menu` site setting may not be overridable via `theme_site_settings` in `about.json` (it is a SiteSetting, not a ThemeSetting). If not overridable, document it as a required one-time admin change: Admin → Settings → Search "top_menu" → set value `hot|latest|categories|top`. Claude's discretion (from CONTEXT.md) covers this determination.
