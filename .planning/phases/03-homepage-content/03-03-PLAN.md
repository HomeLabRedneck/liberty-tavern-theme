Plans 03-01 and 03-02 have been executed:
- `tavern-banner.gjs` now renders `.tavern-banner__stats` (live stats panel) and `.tavern-trending` (trending strip below banner)
- `common/common.scss` now contains all styles for both surfaces, plus `.category-boxes .category-box` extensions for cream card background, left border, border-radius, box-shadow, and hover lift

The only remaining step is activating the `categories_boxes` layout in Discourse so the CSS selectors have DOM elements to target.

Why this cannot be done by code:
The setting `desktop_category_page_style` does not have `themeable: true` in Discourse's `config/site_settings.yml`. Adding it to `theme_site_settings` in `about.json` is silently ignored. There is no CSS-only workaround — the layout class `categories-boxes` on `<body>` is only applied when the admin setting is active.
  </what-built>
  <how-to-verify>
Step 1 — Set the admin setting:
1. Log into Discourse as admin
2. Navigate to Admin → Settings → Interface (or use the search bar and search "desktop category page")
3. Find the setting: "desktop category page style"
4. Set the value to: `categories_boxes with featured topics` or `categories_boxes`
   (Either value activates the categories_boxes DOM layout. Use "categories_boxes with featured topics" if you want the Discourse-native featured topic shown; use "categories_boxes" for a cleaner grid without the featured area.)
5. Save the setting

Step 2 — Verify the layout applied:
1. Navigate to the homepage (the /categories or / route)
2. Right-click on a category entry and inspect the DOM
3. Confirm `<body>` has class `categories-boxes` (or the category container has class `category-boxes`)
4. Confirm individual category entries have class `category-box` (not `category-list-item`)
5. Confirm the room cards display: cream background, oxblood/colored left border, rounded corners, and subtle shadow

Step 3 — Verify the SCSS is taking effect:
1. Open browser DevTools → Elements
2. Select a `.category-box` element
3. In the Styles panel, confirm these rules are applied:
   - `border-radius: 2px` (from Phase 3 §6 extension)
   - `box-shadow: 0 2px 8px rgba(0,0,0,0.07)` (from Phase 3 §6 extension)
   - `border-left: 4px solid var(--category-color, var(--tertiary))` (from existing §6)
   - `background: var(--secondary)` (resolves to cream `#f5ebd9`)

If `.category-box` elements are absent from the DOM, the setting was not saved or is set to a different value. Check Admin → Settings again.
  </how-to-verify>
