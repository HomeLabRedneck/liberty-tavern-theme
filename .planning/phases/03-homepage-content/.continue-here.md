# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: `display: contents` on `.category-box-inner` — caused layout failure. Children didn't respond to `grid-column` rules because DOM nesting is deeper than expected. Use `display: grid` directly on `.category-box-inner` instead.
- [ ] CONSTRAINT: `.wrap` horizontal constraints — only `margin-top`/`padding-top` were zeroed previously, leaving `max-width` and `margin: 0 auto` active. Category-boxes and topic-list rendered narrower than the full-viewport trending strip. Always zero ALL sides.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| `display: contents` for grid passthrough | Applied `display: contents` to `.category-box-inner` expecting children to become grid items — they ignored `grid-column` because DOM is deeper | blocking | Apply `display: grid` directly to the element whose children you need to place |
| Partial wrapper zeroing | Only zeroed `margin-top`/`padding-top` on `.wrap` — left horizontal `max-width` and `margin: 0 auto` active, misaligning right edge | advisory | When removing Discourse layout constraints, always zero `max-width`, `width`, `margin`, AND `padding` for all sides |
