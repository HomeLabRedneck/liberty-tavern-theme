# BLOCKING CONSTRAINTS — Read Before Anything Else

- [ ] CONSTRAINT: `display: grid !important` required on `.category-box-inner` — Discourse sets `display: flex; flex-direction: column` with enough specificity to win without `!important`.
- [ ] CONSTRAINT: `width: auto !important` on `.category-boxes` — Discourse sets `width: 100%`, so `margin-right: 64px` alone causes right overflow.
- [ ] CONSTRAINT: `.wrap` must be fully zeroed (all 4 sides) on homepage routes — partial zeroing (top only) leaves horizontal max-width active.
- [ ] CONSTRAINT: `flex-basis: 100%` on flex items inside a non-explicitly-sized container resolves to `content` not container width — use `width: 100% !important` instead.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| `display: contents` for grid passthrough | DOM is deeper than expected; children ignore grid-column rules | blocking | Apply `display: grid` directly to the element whose children you want as grid items |
| Partial wrapper zeroing | Zeroing only top margin/padding leaves horizontal constraints active | advisory | Always zero all 4 sides: `max-width: none; width: 100%; margin: 0; padding: 0` |
