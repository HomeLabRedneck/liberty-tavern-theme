IMPORTANT CONSTRAINT: CSS `::before` and `::after` render as inline by default. The `display: block` overrides that. However, these pseudo-elements render INSIDE `.category-boxes` as its first and last children — meaning `::after` will appear AFTER the last card, not between the heading and the grid. 

To work around this: use `::before` for BOTH lines by setting `content` to a two-line value with a line break, then applying separate color via a wrapper span — but that is not possible in pure CSS. 

ACTUAL CORRECT APPROACH: Use `.category-boxes::before` for the "The Rooms" heading only (one pseudo-element). Add a SEPARATE parent selector approach for the subtitle. Discourse wraps `.category-boxes` in `.category-list-container` or `.categories-boxes-with-topics`. Target:

```scss
// Gap closure: "The Rooms" section heading above category grid
.category-list-container .category-boxes::before,
.categories-boxes-with-topics .category-boxes::before,
.category-boxes::before {
  content: "The Rooms";
  display: block;
  font-family: var(--font-display);
  font-style: italic;
  font-weight: 900;
  font-size: 28px;
  color: #1C1410;
  line-height: 1.1;
  margin-bottom: 4px;
}
```

For the subtitle, use a SEPARATE `::after` on `.category-boxes` BUT position it correctly. The subtitle must appear between the `::before` heading and the first card. Use CSS `order` with flexbox OR accept that `::after` goes after the last card and instead use a single `::before` with multi-line content using `\a` escape and `white-space: pre-line`:

```scss
// Gap closure: "The Rooms" section heading + subtitle above category grid
// Uses ::before with white-space: pre-line to render two styled lines.
// NOTE: Both lines share the same font style — heading dominates (Playfair italic).
// Subtitle styling via separate ::after is architecturally impossible without flexbox order.
// Decision: render two-line ::before with heading style; subtitle is plain smaller text.
.category-boxes::before {
  content: "The Rooms\a CHOOSE WHERE TO GATHER";
  white-space: pre-line;
  display: block;
  font-family: var(--font-display);
  font-style: italic;
  font-weight: 900;
  font-size: 28px;
  color: #1C1410;
  line-height: 1.3;
  margin-bottom: 16px;
}
```

This results in both lines rendered in Playfair italic — which is acceptable as the gap note says "Fallback: If CSS pseudo-content can't achieve two-line heading, use a single heading line only."

FINAL DECISION for this task: Use the single-line fallback approach for the heading only (cleaner, no CSS hack). Write:

// Subtitle rendered via ::after — appears after all cards inside grid.
