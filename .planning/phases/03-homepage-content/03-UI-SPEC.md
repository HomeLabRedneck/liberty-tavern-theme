# Phase 3 — UI Design Contract: Homepage Content

> Visual and interaction contract for Phase 3. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Covers three new visual surfaces: stats panel, trending strip (standalone cream section), and room cards.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (pure Discourse theme SCSS) |
| Preset | not applicable |
| Component library | none — Glimmer `.gjs` components with SCSS BEM classes |
| Icon library | Unicode emoji inline (flame `🔥` for trending header) |
| Font — Display | Playfair Display (loaded via `head_tag.html`) |
| Font — Serif | Spectral (loaded via `head_tag.html`) |
| Font — UI | Inter (loaded via `head_tag.html`) |

Note: No `@import` in SCSS. Fonts are loaded exactly once via `<link>` in `common/head_tag.html` per CLAUDE.md Rule 3.

## Spacing Scale

All values are multiples of 4. Existing tokens from `common.scss` are preserved; Phase 3 adds new usages.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps within stat rows, badge circle offsets |
| sm | 8px | Gap between stat label and divider rule; trending card internal padding |
| md | 16px | Stats panel internal horizontal padding; trending card gap row |
| lg | 24px | Stats panel vertical padding (top/bottom); trending section horizontal padding; gap between trending cards |
| xl | 32px | Stats panel border offset from banner edges; trending section bottom padding |
| 2xl | 48px | Trending section top padding (from banner bottom border to section content) |
| 3xl | 64px | Inherited from banner — not used in new surfaces |

Exceptions:
- Stats panel corner brackets: 10px arm length, 10px arm width, positioned 8px inset from panel corners. This is a decorative measurement derived from visual calibration against the existing 1px brass border on `&__feature`, not a spacing token.
- Trending section vertical padding: 32px top (breathing room below banner's brass border) / 24px bottom. Asymmetric because the section flows into the category grid below.
- Category box card padding: inherit from Discourse's `.category-boxes` internal layout; only border and color overrides are applied, not padding resets.

## Component Specifications

### Surface 1: Stats Panel (`.tavern-banner__stats`)

Replaces `.tavern-banner__feature` and `.tavern-banner__badges` entirely. Lives in `<aside class="tavern-banner__aside">` inside the dark banner grid.

**Container — `.tavern-banner__stats`**

| Property | Value |
|----------|-------|

| Border | `1px solid var(--tavern-brass)` (matches removed `&__feature` border) |

**Corner bracket decoration — `::before` and `::after`**

| Property | `::before` (top-left) | `::after` (bottom-right) |
|----------|-----------------------|--------------------------|

| Width / Height | `10px / 10px` | `10px / 10px` |
| Border | `border-top: 2px solid var(--tavern-brass); border-left: 2px solid var(--tavern-brass)` | `border-bottom: 2px solid var(--tavern-brass); border-right: 2px solid var(--tavern-brass)` |

Note: These sit on top of the container's existing `1px solid var(--tavern-brass)` border. They create a double-corner mark effect matching Image 4. The `::before`/`::after` overlay the border at exactly the corners, 1px inset from the container edge so the 2px bracket lines visually extend the corner.

Content: "TONIGHT AT THE HOUSE"

**Loading state**

While `this.loading` is true: render four rows with `—` as the number value. Use `{{#unless this.loading}}` guard on the number span only, not the entire panel, so the panel chrome (heading, labels) appears immediately.

```
{{/unless}}
```

## Interaction Contract

### Stats panel

- No interactivity. Static display of four numbers. No hover, click, or animation states.
- Loading state: render panel chrome (heading + 4 label rows) immediately; populate numbers after `Promise.all` resolves. Numbers show "—" while loading.

### Trending strip

- Three topic title links are standard `<a href>` — rely on Discourse's router. No JS action needed.
- "ALL HOT THREADS →" is a standard `<a href="/hot">` — no JS action.
- No hover card expansion, no carousel, no infinite scroll.
- Section is conditionally rendered: `{{#if this.shouldShow}}` (same `shouldShow` getter as the banner — only renders on `discovery.*` routes).

### Room cards

- All interactivity is provided by Discourse's native `categories_boxes` rendering. Phase 3 adds only SCSS — no JS changes to category routing or card click behavior.
- Hover state: `box-shadow` lift on `.category-box` (declared above).
